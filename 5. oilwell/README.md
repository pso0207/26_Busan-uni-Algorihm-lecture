# 5번 문제 Oil Well (유전 찾기)

이 문제는 주어진 1차원 공간(크기 $N$)에서 숨겨진 유전의 시작점($L$)과 끝점($R$)을 찾는 최적화 문제입니다.

## 1. 문제 개요
- **주어진 정보**: 전체 구간의 크기 $N$
- **목표**: 유전이 매장된 정확한 구간 $[L, R]$을 찾아내는 것.
- **제약 조건**: `observe(i, j)` 함수를 최대 35회까지만 호출할 수 있습니다. (이분 탐색의 효율성이 중요)

## 2. 주요 함수 설명 (`oil.h`)
- `oil_size()`: 전체 구간의 크기 $N$을 반환합니다.
- `observe(i, j)`: $[i, j]$ 구간을 조사합니다.
  - **0**: 구간 내에 유전이 전혀 없음.
  - **1**: 구간 전체가 유전으로 덮여 있음 ($[i, j] \subseteq [L, R]$).
  - **2**: 구간의 일부에만 유전이 포함되어 있음.
- `oil_report(l, r)`: 최종적으로 찾은 $L, R$ 값을 제출합니다.

## 3. 알고리즘 설계 (이분 탐색)
전체 구간 $N$이 클 경우(예: $10^5$), 선형 탐색으로는 35회 이내에 답을 찾을 수 없습니다. 따라서 $O(\log N)$의 복잡도를 가지는 이분 탐색(Binary Search)을 두 번 수행합니다.

### 시작점($L$) 찾기
- 구간 $[low, high]$의 중간값 `mid`를 기준으로 `observe(low, mid)`를 호출합니다.
- 결과가 **0**이면 유전은 `mid`보다 오른쪽에 있으므로 $low = mid + 1$로 좁힙니다.
- 결과가 **0이 아니면**(1 또는 2), 유전의 시작점 $L$은 `mid` 이하에 있으므로 $high = mid$로 좁힙니다.

### 끝점($R$) 찾기
- 구간 $[low, high]$의 중간값 `mid`를 기준으로 `observe(mid + 1, high)`를 호출합니다.
- 결과가 **0**이면 유전은 `mid` 이하에 있으므로 $high = mid$로 좁힙니다.
- 결과가 **0이 아니면**, 유전의 끝점 $R$은 `mid + 1` 이상에 있으므로 $low = mid + 1$로 좁힙니다.

## 4. 복잡도 및 효율성
- 각 이분 탐색은 약 $\log_2 N$회의 쿼리를 사용합니다.
- $N = 10^5$일 때 $\log_2(10^5) \approx 17$이므로, 두 번의 탐색을 합쳐 약 34~35회 이내에 정확한 범위를 찾아낼 수 있습니다.
