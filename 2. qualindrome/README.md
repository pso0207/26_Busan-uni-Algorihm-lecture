# Qualindrome (회문과 유사 회문 판별)

문자열이 주어졌을 때, 해당 문자열이 **회문(Palindrome)**인지, 한 글자를 지워서 회문이 될 수 있는 **유사 회문(Pseudo-palindrome)**인지, 혹은 둘 다 아닌지를 판별하는 알고리즘입니다.

## 🚀 문제 정의

- **회문 (Type 1)**: 문자열의 앞뒤가 똑같은 경우. (예: `level`, `aba`)
- **유사 회문 (Type 2)**: 한 문자를 삭제했을 때 회문이 되는 경우. (예: `summuus`에서 하나를 지우면 `summus`)
- **일반 문자열 (Type 3)**: 위 두 조건에 모두 해당하지 않는 경우.

## 🛠 알고리즘: 투 포인터 (Two-Pointer)

이 문제는 **투 포인터** 기법을 사용하여 효율적으로 해결할 수 있습니다.

1. 문자열의 양 끝을 가리키는 `left`, `right` 포인터를 설정합니다.
2. 두 문자가 같으면 포인터를 중앙으로 좁힙니다.
3. 두 문자가 다를 경우:
   - **왼쪽 문자를 건너뛰었을 때** 남은 부분이 회문인지 확인합니다.
   - **오른쪽 문자를 건너뛰었을 때** 남은 부분이 회문인지 확인합니다.
   - 둘 중 하나라도 회문이면 **유사 회문(Type 2)**입니다.
   - 둘 다 회문이 아니면 **일반 문자열(Type 3)**입니다.
4. 불일치 없이 탐색을 마치면 **회문(Type 1)**입니다.

## ⏱ 시간 복잡도

- **O(N)**: 문자열의 길이를 $N$이라고 할 때, 문자열을 한 번 스캔하며 불일치 시 최대 두 번의 추가 부분 스캔이 발생하므로 선형 시간에 동작합니다.
