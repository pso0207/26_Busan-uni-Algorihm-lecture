# Container (2등 찾기 알고리즘)

주어진 $N$개의 요소 중에서 비교 횟수를 최소화하여 **두 번째로 큰 값(Second-largest element)**을 찾는 최적화 알고리즘입니다.

## 🚀 문제 정의

단순하게 정렬을 하거나 전체를 두 번 탐색하는 방식보다 효율적으로 2등을 찾아야 합니다.

- **최소 비교 목표**: $N + \log_2 N - 2$ 번 이하의 비교.
- **제약**: `box_comp` 함수를 통해 두 요소를 비교하며, 호출 횟수에 제한이 있음.

## 🛠 알고리즘: 토너먼트 & 히스토리 (Tournament Method)

이 문제는 **토너먼트 알고리즘**을 사용하여 비교 횟수를 최적화합니다.

1. **토너먼트 진행**: 모든 요소를 1:1로 비교하며 승자를 결정합니다. 최종 승자(1등)가 결정될 때까지 반복합니다.
2. **패배 기록 (History)**: 각 대결에서 승자는 자신이 이긴 상대를 자신의 `history` 리스트에 저장합니다.
3. **2등 후보군 추출**: 전체 1등에게 직접 패배한 요소들만이 2등이 될 가능성이 있습니다.
   - 1등은 총 $\lceil \log_2 N \rceil$번의 경기를 치렀으므로, 후보군은 최대 $\log_2 N$개입니다.
4. **최종 2등 판별**: 추출된 후보군들 중에서 가장 큰 값을 찾으면, 그 값이 전체에서의 2등이 됩니다.

## ⏱ 시간 복잡도 및 비교 횟수

- **비교 횟수**: $(N-1) + (\lceil \log_2 N \rceil - 1)$
- **공간 복잡도**: $O(N \log N)$ (히스토리 저장을 위한 벡터 배열 사용)
